Basic Git Tutorial

Let's get started with Git! Below are some useful commands to help you manage your project.

1. Set Up Your Identity

Before diving into Git, let's tell it who you are. Use these commands to set up your name and email. It's like signing your name on your work!

$ git config --global user.name "Thierry Demolder"
$ git config --global user.email "tdemolder@gmail.com"

2. Set Default Branch Name

Git recently changed the name of the main branch from "master" to "main" to be more inclusive. Let's make sure Git knows about it:

$ git config --global init.default branch main

3. Navigate to Your Project Directory

Use the cd command to go to the main folder of your project. This is where all your project files live.

$ cd path/to/your/project

4. Initialize Your Repository

Now that you're in your project folder, let's initialize a Git repository. It's like telling Git, "Hey, keep an eye on these files!"

$ git init

5. Add Files to the Staging Area

You've got all these files in your project folder. Let's tell Git to start tracking changes to them.

$ git add index.htm     # Add a specific file
$ git add .             # Add all files in the directory
$ git add --all         # Add all files, including deleted ones
$ git -A                # Shorthand for adding all files

6. Manipulate Your Files

Now, let's manipulate our files a bit. We'll learn how to erase, restore, track, and untrack them.

  • To delete a file: rm filename
  • To restore a deleted file: git checkout -- filename
  • To track a new file: git add filename
  • To untrack a file: git reset HEAD filename
  • Checking your current situation git status
  • Unstage a filegit rm --cached
  • Commit your filesgit commit -m "reason of the commit"
  • Check the difference between the former file and the new modified one git diff

7. Create a .gitignore File

Sometimes, there are files you don't want Git to track, like temporary files or sensitive information. Create a .gitignore file and list those files there.

$ touch .gitignore     # Create a .gitignore file

8. Check Your Commit History (Logs)

It's good to keep track of what you've done. Let's check the commit history.

$ git log
$ git log --oneline

9. Take a Snapshot of Your Repository

You've made some changes, and you want to save them. Let's take a snapshot (commit) of your project.

$ git commit -m "first commit - committing all files to the repository"

10. Link Your Local Repository to GitHub

Time to share your project with the world! We'll link your local repository to a remote one on GitHub.

$ git remote add origin https://github.com/ThierryDemolder/Vins-Deconinck.git

11. Rename Your Branch (Optional)

Sometimes, it's necessary to rename branches. In this case, let's rename our main branch to make sure it matches with GitHub.

$ git branch -M main

12. Push Your Changes to GitHub

Finally, let's send your project up to GitHub so everyone can see your awesome work!

$ git push -u origin main
